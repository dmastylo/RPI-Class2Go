We Get Paid to Build Open Source Software

We began building Class2Go this past summer and now now have hordes of students taking Stanford classes for free.

Our stack is modern and sane: Python/Django and MySQL on the backend, Bootstrap, jQuery, Popcorn (and others) on the frontend. We bring up machines quickly in AWS with Chef.

All our source is on GitHub. Our bug list is there too. It's fun to work in a small team that can move fast — we ship weekly (at least!). And the Stanford campus is a cool place to work.

Local (or willing to re-locate) applicants only please.

Python Engineer

We are looking for an engineer to work on core features for Class2Go. Nearly everything we do is in Python/Django. Not only can we get a lot done ourselves in Python, but there's a wonderful ecosystem of tools and features that we love to use.

You should be:

  • Fluent in Python with at least three years hands-on experience. Experience with Django is a plus. Point us to something you've done that you're proud of.
  • Comfortable developing software with a distributed team. You love Git and GitHub.
  • A full-stack engineer able and willing to get your hands dirty. Install scripts or front-end code shoudn't faze you.

DevOps and Release

This engineer should be passionate about reliable and scalable cloud systems. Do you know the right way to build things, but never gotten the chance to do it right?

They will also be the team's release engineer. We need someone to set up code management and process for a codebase used by universities across the world.

We are looking for three skills:

  • Operations: proven ability to operate production cloud systems. You've lived in the Amazon stack. You have an informed opinion about Puppet v. Chef (we use chef).
  • Release: you live and breathe git. You eat merge conflicts for breakfast.
  • Development: why do something twice when you can automate it? Python is our tool of choice, but you should be super comfortable at the Unix command line.
